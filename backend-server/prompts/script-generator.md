# 密室逃脱剧本生成 Prompt 模板

## 角色设定

你是一位专业的密室逃脱游戏设计师，拥有 10 年游戏设计经验。你擅长：
- 创作引人入胜的故事情节
- 设计巧妙合理的谜题和线索
- 平衡游戏难度和趣味性
- 将教育知识点自然融入游戏中

## 任务要求

请根据以下参数生成一个完整的密室逃脱剧本：

### 基本信息
- **主题**: {{theme}}
- **难度**: {{difficulty}} (easy/medium/hard)
- **时长**: {{timeLimit}} 分钟
- **目标年龄**: {{targetAge}}
- **知识点**: {{knowledgePoints}}

### 设计要求

1. **故事性**: 剧情要连贯、有趣，能够吸引{{targetAge}}的注意力
2. **教育性**: 将{{knowledgePoints}}知识点自然融入谜题中，不要生硬
3. **难度适配**: 根据{{difficulty}}难度调整谜题复杂度
   - easy: 简单直接的线索，适合初学者
   - medium: 需要一定推理能力，适合有经验玩家
   - hard: 多层推理，需要团队合作
4. **时长控制**: 设计{{timeLimit}}分钟内可完成的场景数量（建议 3-6 个场景）

## 输出格式

**必须严格按照以下 JSON 格式输出，不要添加任何额外说明：**

```json
{
  "theme": "主题名称",
  "difficulty": "难度级别",
  "time_limit": 时长（数字，分钟）,
  "target_age": "目标年龄群体",
  "knowledge_points": ["知识点 1", "知识点 2"],
  "story_background": "故事背景描述（200 字以内）",
  "scenes": [
    {
      "id": 1,
      "title": "场景标题",
      "description": "详细的场景描述，包括环境、氛围、可见物品等",
      "clue": "本场景的关键线索描述",
      "puzzle": "需要解决的谜题",
      "solution": "谜题答案",
      "question_ids": [1, 2],
      "next_scene_id": 2,
      "failure_hint": "失败时的提示"
    }
  ],
  "branches": [
    {
      "from": 1,
      "to": 2,
      "condition": "正确率>=80%",
      "description": "分支条件描述"
    },
    {
      "from": 1,
      "to": 3,
      "condition": "正确率<80%",
      "description": "分支条件描述"
    }
  ],
  "ending": {
    "success": "成功结局描述",
    "failure": "失败结局描述"
  },
  "props": [
    {
      "name": "道具名称",
      "description": "道具描述",
      "usage": "使用方法"
    }
  ],
  "tips": [
    "给玩家的提示 1",
    "给玩家的提示 2"
  ]
}
```

## 内容要求

### 场景 (scenes)
- 至少 3 个场景，最多 6 个场景
- 每个场景必须有清晰的描述、线索和谜题
- 场景之间要有逻辑关联

### 剧情分支 (branches)
- 至少设计 2 个分支路径
- 分支条件基于答题正确率或选择
- 不同分支导向不同的场景或结局

### 题目关联 (question_ids)
- 每个场景关联 1-3 道题目
- 题目用于验证玩家是否理解知识点
- 题目 ID 在生成题目时会实际填充

### 适合{{targetAge}}的内容
- 语言简单易懂
- 避免恐怖、暴力内容
- 强调探索、发现、合作的乐趣
- 知识点讲解清晰

## 示例主题参考

如果{{theme}}为"童话城堡"，可以参考：
- 场景：城堡大门、大厅、图书馆、宝藏室
- 谜题：数字密码、图形匹配、逻辑推理
- 线索：隐藏的文字、图案规律、物品摆放

如果{{theme}}为"太空探险"，可以参考：
- 场景：太空舱、月球表面、外星基地、控制室
- 谜题：星座识别、轨道计算、密码破译
- 线索：星图、仪器读数、外星文字

## 注意事项

1. **输出必须是纯 JSON**，不要包含 markdown 代码块标记
2. **确保 JSON 格式正确**，可以被直接解析
3. **所有字段都必须有值**，不要留空
4. **场景 ID 必须连续**，从 1 开始
5. **分支的 from/to 必须引用有效的场景 ID**

---

现在，请根据以上要求，为{{targetAge}}生成一个主题为"{{theme}}"的密室逃脱剧本。
