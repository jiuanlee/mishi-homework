# 端到端测试用例 - 完整流程

## 说明

端到端测试验证完整的用户流程，从开始到结束，涉及多个页面和接口。

---

## E2E-001: 老师生成作业完整流程

| 项目 | 内容 |
|------|------|
| **用例编号** | E2E-001 |
| **流程名称** | 老师生成并发布作业 |
| **优先级** | P0 |
| **涉及角色** | 老师 |
| **前置条件** | - 老师账号已注册并登录<br>- 已创建班级并添加学生 |
| **测试步骤** | 1. 老师登录 Web 端<br>2. 进入"作业管理"<br>3. 点击"创建作业"<br>4. 填写作业信息 (标题、学科、年级)<br>5. 选择知识点 (至少 2 个)<br>6. 选择题型 (选择题 + 填空题)<br>7. 设置难度 (中等)<br>8. 设置题目数量 (10 题)<br>9. 设置时间限制 (30 分钟)<br>10. 点击"生成作业"<br>11. 预览作业内容<br>12. 调整不满意的题目<br>13. 点击"发布"<br>14. 选择目标班级<br>15. 设置截止日期 (3 天后)<br>16. 确认发布 |
| **预期结果** | - 作业成功创建<br>- 作业发布到班级<br>- 学生收到通知<br>- 作业出现在学生作业列表 |
| **验证点** | - 数据库中存在作业记录<br>- 学生端可见作业<br>- 通知已发送 |
| **执行状态** | TODO |
