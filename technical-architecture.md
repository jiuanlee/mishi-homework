# 技术架构文档 - 寓教于乐·密室做题家

## 1. 产品概述

- **产品名称**：寓教于乐·密室做题家
- **定位**：小学课后作业游戏化解决方案
- **核心功能**：密室逃脱/剧本杀 + 课后作业
- **目标用户**：小学老师（生成作业）、学生（游戏化做题）

---

## 2. 技术栈选型

### 2.1 前端技术栈

#### 老师端（Web）
| 技术 | 选型 | 理由 |
|------|------|------|
| 框架 | Vue 3 + TypeScript | 生态成熟，学习曲线平缓，适合后台管理系统 |
| UI 组件库 | Element Plus | 丰富的后台组件，主题定制方便 |
| 状态管理 | Pinia | Vue 3 官方推荐，轻量易用 |
| 构建工具 | Vite | 快速开发体验，热更新 |
| 路由 | Vue Router 4 | 官方路由方案 |

#### 学生端（小程序）
| 技术 | 选型 | 理由 |
|------|------|------|
| 框架 | uni-app | 一套代码多端发布（微信/支付宝小程序） |
| UI 组件库 | uView UI | 专为 uni-app 设计的组件库 |
| 状态管理 | Pinia | 与老师端保持一致 |

### 2.2 后端技术栈

| 技术 | 选型 | 理由 |
|------|------|------|
| 语言 | Node.js (TypeScript) | 前后端语言统一，开发效率高 |
| 框架 | NestJS | 企业级框架，模块化，支持微服务演进 |
| API 规范 | RESTful + OpenAPI | 标准化接口，自动生成文档 |
| 认证 | JWT | 无状态认证，适合多端场景 |

### 2.3 数据库

| 类型 | 选型 | 用途 |
|------|------|------|
| 关系型 | PostgreSQL 15 | 用户、作业、题目等核心数据 |
| 缓存 | Redis 7 | 会话缓存、排行榜、热点数据 |
| 对象存储 | 阿里云 OSS / 腾讯云 COS | 图片、音频、视频等多媒体资源 |

### 2.4 部署与运维

| 技术 | 选型 | 理由 |
|------|------|------|
| 容器化 | Docker + Docker Compose | 环境一致性，快速部署 |
| 云服务 | 阿里云 / 腾讯云 | 国内访问速度快，生态完善 |
| CI/CD | GitHub Actions / 云效 | 自动化构建部署 |
| 监控 | Prometheus + Grafana | 系统监控与告警 |
| 日志 | ELK Stack | 日志收集与分析 |

### 3.2 模块划分

#### 核心业务模块

| 模块 | 职责 | 主要功能 |
|------|------|----------|
| 用户模块 | 用户管理 | 注册登录、角色管理、个人信息 |
| 知识点模块 | 知识体系 | 知识点树、难度等级、关联关系 |
| 题目模块 | 题库管理 | 题目 CRUD、题型、解析、多媒体 |
| 作业模块 | 作业生命周期 | 作业生成、发布、批改、反馈 |
| 游戏化模块 | 游戏引擎 | 密室场景、剧情分支、奖励机制 |
| 统计模块 | 数据分析 | 学习进度、正确率、薄弱点分析 |

#### 支撑模块

| 模块 | 职责 | 主要功能 |
|------|------|----------|
| 认证模块 | 安全认证 | JWT 签发验证、权限控制 |
| 通知模块 | 消息推送 | 模板消息、作业提醒 |
| 文件模块 | 资源管理 | 上传下载、CDN 加速 |

### 3.3 数据流向

#### 作业生成流程
```
老师端 → 选择知识点 → 选择题型/难度 → 生成作业 → 发布
   │                                              │
   ▼                                              ▼
API: /api/assignments/create              API: /api/assignments/publish
   │                                              │
   ▼                                              ▼
后端验证 → 组题算法 → 存储作业 → 推送通知 → 学生可见
```

#### 学生做题流程
```
学生端 → 进入密室场景 → 解锁题目 → 答题 → 提交
   │                                    │
   ▼                                    ▼
API: /api/assignments/:id        API: /api/answers/submit
   │                                    │
   ▼                                    ▼
获取作业数据 → 渲染场景 → 验证答案 → 更新进度 → 奖励反馈
```

### 4.3 Docker Compose 配置示例

```yaml
version: '3.8'
services:
  api:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/m密室
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db
      - redis

  db:
    image: postgres:15
```

## 5. 技术决策说明

### 5.1 为什么选择 NestJS？
- **模块化架构**：天然支持业务模块划分
- **TypeScript 优先**：类型安全，减少运行时错误
- **微服务就绪**：未来可平滑演进为微服务架构
- **生态丰富**：官方支持 GraphQL、微服务、WebSocket 等

### 5.2 为什么选择 uni-app？
- **多端发布**：一套代码发布微信/支付宝小程序
- **学习成本低**：Vue 语法，团队上手快
- **性能优秀**：原生渲染，接近原生体验

### 5.3 为什么选择 PostgreSQL？
- **JSONB 支持**：灵活存储题目元数据
- **全文检索**：内置全文搜索能力
- **扩展性强**：支持地理信息、时序数据等扩展

## 6. 扩展性考虑

### 6.1 短期（MVP）
- 单体架构，快速迭代验证
- 重点打磨核心功能：作业生成 + 游戏化做题

### 6.2 中期（增长期）
- 读写分离，数据库优化
- 引入消息队列（RabbitMQ）处理异步任务
- CDN 加速静态资源

### 6.3 长期（规模化）
- 微服务拆分（用户服务、作业服务、游戏服务）
- 多租户支持（学校/机构独立部署）
- AI 组题引擎（基于大模型的智能组题）
