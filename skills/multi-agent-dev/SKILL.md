# 多 Agent 协作开发 Skill

你是一个**协调 Agent**，负责协调多个专业 Agent 完成开发任务。

## 核心能力

- ✅ 分析任务需求
- ✅ 协调专业 Agent
- ✅ 汇总执行结果
- ✅ 生成完整报告

## 可用 Agent 角色

| Agent | 职责 | 系统提示词 |
|-------|------|-----------|
| **规划 Agent** | 分析需求，制定计划 | "你是资深技术规划师，擅长拆解复杂需求为可执行任务" |
| **编码 Agent** | 编写高质量代码 | "你是资深开发工程师，擅长编写清晰、可维护的代码" |
| **测试 Agent** | 生成测试并运行 | "你是资深测试工程师，擅长设计全面的测试用例" |
| **审查 Agent** | 代码审查和评估 | "你是资深代码审查专家，擅长发现代码问题和安全隐患" |

## 工作流程

```
用户任务
   ↓
协调 Agent（分析需求）
   ↓
┌──────────────────────────────────────┐
│ 1. 规划 Agent → 需求分析和任务拆解    │
│ 2. 编码 Agent → 根据计划编写代码      │
│ 3. 测试 Agent → 生成测试并运行        │
│ 4. 审查 Agent → 代码审查和评估        │
└──────────────────────────────────────┘
   ↓
协调 Agent（汇总结果）
   ↓
返回用户
```

## 调用格式

### 调用单个 Agent

```
@规划 Agent [任务描述]
@编码 Agent [任务描述]
@测试 Agent [任务描述]
@审查 Agent [任务描述]
```

### 完整协作流程

```
启动多 Agent 协作流程

任务：[任务描述]

要求：
1. 规划 Agent 分析需求，输出任务列表
2. 编码 Agent 根据计划编写代码
3. 测试 Agent 生成测试并运行
4. 审查 Agent 审查代码质量
5. 汇总所有结果返回
```

## 使用示例

### 示例 1：开发新功能

```
启动多 Agent 协作

任务：开发一个用户登录功能

要求：
- 支持邮箱和密码登录
- 密码强度检查
- 登录失败次数限制
- Session 管理

请按流程协调各个 Agent 完成。
```

### 示例 2：修复 Bug

```
启动多 Agent 协作

任务：修复用户无法登出的问题

问题描述：
用户点击登出按钮后，Session 未清除，仍可访问受保护页面。

要求：
1. 编码 Agent 分析问题并修复
2. 测试 Agent 验证修复
3. 审查 Agent 审查代码
```

### 示例 3：代码重构

```
启动多 Agent 协作

任务：重构用户管理模块

当前问题：
- 代码重复严重
- 函数过长（>100 行）
- 缺少注释

要求：
1. 规划 Agent 制定重构计划
2. 编码 Agent 执行重构
3. 测试 Agent 确保功能正常
4. 审查 Agent 评估重构质量
```

## 配置选项

### 启用/禁用 Agent

```yaml
agents:
  planning: true    # 启用规划 Agent
  coding: true      # 启用编码 Agent
  testing: true     # 启用测试 Agent
  review: true      # 启用审查 Agent
```

### 并行执行

```yaml
execution:
  parallel: false   # 是否并行执行（实验性）
  timeout: 3600     # 超时时间（秒）
```

### 输出选项

```yaml
output:
  save_reports: true      # 保存报告到文件
  report_dir: docs/       # 报告保存目录
  verbose: false          # 详细输出模式
```

## 错误处理

### Agent 调用失败

如果某个 Agent 调用失败：

1. 重试 1 次
2. 如果仍然失败，记录错误
3. 继续执行后续 Agent（如果可能）
4. 在汇总报告中标注失败环节

### 超时处理

如果某个 Agent 执行超时：

1. 终止该 Agent 执行
2. 记录超时错误
3. 评估是否可以跳过
4. 在汇总报告中说明

## 最佳实践

### 1. 明确任务描述

❌ 不好：
```
开发一个功能
```

✅ 好：
```
开发一个用户注册功能，包含：
- 邮箱验证
- 密码强度检查（至少 8 位，包含大小写和数字）
- 用户信息存储到 PostgreSQL
- 发送欢迎邮件
```

### 2. 提供上下文

❌ 不好：
```
修复这个 bug
```

✅ 好：
```
修复 bug：用户登录后，刷新页面会退出登录

上下文：
- 使用 JWT Token 认证
- Token 存储在 localStorage
- 后端验证中间件已实现

相关文件：
- src/auth/login.py
- src/middleware/auth.py
```

### 3. 指定输出要求

❌ 不好：
```
写代码
```

✅ 好：
```
编写代码，要求：
- 遵循 PEP 8 规范
- 添加类型注解
- 函数必须有 docstring
- 单元测试覆盖率 > 80%
```

## 常见问题

### Q1: 如何跳过某个 Agent？

在任务描述中说明：

```
启动多 Agent 协作

任务：[任务描述]

跳过：审查 Agent（时间紧急）
```

### Q2: 如何并行执行？

目前不支持真正的并行执行，但可以优化流程：

```
1. 规划 Agent 完成后
2. 编码 Agent 和测试 Agent 同时准备
3. 编码完成后立即运行测试
```

### Q3: 如何保存中间结果？

所有 Agent 的输出会自动保存到：

```
~/.openclaw/workspace/skills/multi-agent-dev/reports/
├── planning-20260316-001.md
├── coding-20260316-001.md
├── testing-20260316-001.md
└── review-20260316-001.md
```
