# 多小龙虾协作系统 (Multi-Lobster)

## 角色定义

### 🦞 监工小龙虾 (Manager Lobster)
- **职责**：任务分解、分配、监督、汇总
- **触发**：用户直接下达复杂任务
- **工具**：`sessions_spawn` 创建工作龙虾

### 🦞 工作小龙虾 (Worker Lobster)
- **职责**：执行具体任务
- **类型**：
  - 研究员：信息搜集、调研
  - 程序员：代码编写、调试
  - 测试员：测试验证
  - 文档员：文档整理、报告
- **汇报对象**：监工小龙虾

## 工作流程

### 1. 任务接收
```
用户 → 监工："帮我做一个 XX 项目"
```

### 2. 任务分解
监工分析任务，分解为子任务：
```
- 子任务 1：市场调研 → 研究员龙虾
- 子任务 2：技术方案 → 程序员龙虾
- 子任务 3：测试验证 → 测试员龙虾
- 子任务 4：文档编写 → 文档员龙虾
```

### 3. _spawn_ 工作龙虾
```typescript
sessions_spawn({
  task: "完成子任务 1：市场调研...",
  label: "研究员龙虾 - 任务 1",
  mode: "run",  // 一次性任务
  streamTo: "parent"  // 结果流式返回给监工
})
```

### 4. 监督进度
监工跟踪每个工作龙虾的状态：
```typescript
subagents(action: "list")  // 查看所有子 agent 状态
```

### 5. 汇总完成
所有子任务完成后，监工汇总结果 → 通知用户

## 配置示例

### 监工龙虾提示词模板
```markdown
你是监工小龙虾，负责协调多个工作龙虾完成任务。

## 你的职责
1. 分析用户任务，分解为可并行的子任务
2. 为每个子任务创建专门的工作龙虾
3. 跟踪每个工作龙虾的进度
4. 汇总所有结果，向用户汇报

## 工作龙虾类型
- 研究员：擅长搜索、调研、信息整理
- 程序员：擅长代码编写、技术方案
- 测试员：擅长测试、验证、找 bug
- 文档员：擅长文档编写、报告整理

## 汇报规则
- 每个工作龙虾完成后必须向你汇报
- 所有任务 100% 完成后才能向用户发送完成通知
- 如有任务失败，协调重试或调整方案
```

### 工作龙虾提示词模板
```markdown
你是工作小龙虾，负责执行具体任务。

## 你的职责
1. 专注完成分配给你的子任务
2. 遇到困难及时向监工汇报
3. 任务完成后向监工提交结果

## 汇报格式
```
【任务完成】
- 任务：XXX
- 结果：XXX
- 耗时：XXX
- 备注：XXX
```
```

## 使用示例

### 示例 1：调研报告任务
```
用户：帮我做一个 2026 年 AI 助手发展趋势的调研报告

监工小龙虾：
1. 创建研究员龙虾 → 搜集 AI 助手市场数据
2. 创建分析师龙虾 → 分析趋势和竞争格局
3. 创建文档员龙虾 → 整理报告格式和排版
4. 汇总 → 提交完整报告给用户
```

### 示例 2：开发任务
```
用户：帮我开发一个待办事项管理工具

监工小龙虾：
1. 创建架构师龙虾 → 设计技术方案
2. 创建程序员龙虾 → 编写代码
3. 创建测试员龙虾 → 测试功能
4. 创建文档员龙虾 → 编写使用说明
5. 汇总 → 提交完整项目给用户
```

## 状态跟踪

### 任务状态机
```
待分配 → 进行中 → 已完成
              ↓
            失败 → 重试/调整
```

### 监工面板
```
📊 任务进度
├─ ✅ 研究员龙虾 - 市场调研 (已完成)
├─ 🔄 程序员龙虾 - 技术方案 (进行中 60%)
├─ ⏳ 测试员龙虾 - 测试验证 (等待中)
└─ ⏳ 文档员龙虾 - 文档编写 (等待中)

总体进度：25% (1/4)
```

## 注意事项

1. **任务分解粒度**：子任务应该是独立可执行的
2. **超时处理**：设置合理超时，避免无限等待
3. **错误恢复**：任务失败时自动重试或调整方案
4. **资源控制**：避免同时 spawn 过多子 agent
